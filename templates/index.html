    • Machine Learning Supervisado

    Google Cloud define el machine learning supervisado como “una categoría del aprendizaje automático que utiliza conjuntos de datos etiquetados para entrenar algoritmos con el objetivo de predecir resultados y reconocer patrones” (s.f.). Estos modelos nos permiten hacer predicciones precisas y es gracias a eso que se usan en varios sectores, como salud, marketing o finanzas.

    ”La salida de la función puede ser un valor numérico (como en los problemas de regresión) o una etiqueta de clase (como en los de clasificación).” (Wikipedia, s.f.). IBM (s.f.) también define al aprendizaje supervisado como un entrenamiento con datos para entender la relación entre su entrada (input) y su salida (output), esto quiere decir que hablamos de funciones aplicados con un algoritmo que procesa estos datos convirtiéndolos en un resultado deseado.

    Categorías

    Generalmente estos modelos de aprendizaje se dividen en dos categorías:

    Clasificación: “Los algoritmos de clasificación se usan para agrupar datos mediante la predicción de una etiqueta categórica o una variable de salida en función de los datos de entrada. La clasificación se utiliza cuando las variables de salida son categóricas, es decir, cuando hay dos o más clases.” (Google Cloud, s.f.).

    Regresión: “los algoritmos de regresión utilizan conjuntos de datos de entrenamiento etiquetados para identificar una relación que se ajuste mejor entre entradas y salidas, de modo que se puedan hacer predicciones matemáticas para nuevas entradas.” (Oracle, 2024).

    ¿Cómo funciona?

    IBM indica el proceso que por lo general se sigue en un modelo de aprendizaje supervisado:

    • Identificar el tipo de dato que será utilizado para entrenar el modelo
    • Acomodar los datos y etiquetarlos de manera que sean “datos etiquetados”, además de eso tienen que ser datos no sesgados para evitar problemas de rendimiento y resultados.
    • Estos datos deben estar divididos en 3: datos de entrenamiento, validación y evaluación.
    • Escoger un algoritmo de machine learning para crear el modelo
    • Entrenar el modelo con el algoritmo escogido
    • Ir validando y evaluando el modelo
    • Asegurar un mantenimiento constante sobre su rendimiento y precisión de los resultados

    Diferencia entre Supervisado y no supervisado

    La mejor forma de remarcar las diferencias entre ambas, es la manera en la que estas se entrenan, como se dijo antes, el modelo supervisado busca producir resultados que el entrenador ya sabe, son patrones ya conocidos, GeekForGeeks (2025) menciona que en cambio, los no supervisados funciona de manera que los datos no están etiquetados, es decir, son datos de los que no le damos un output a validar, contrario al supervisado

    El objetivo del no supervisado es encontrar patrones que no son observables a simple vista, diferente al supervisado en el que ya conocemos su output y validamos los resultados del modelo en base a los datos que ya sabemos.
